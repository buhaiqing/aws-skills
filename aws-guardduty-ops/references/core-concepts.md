# AWS GuardDuty Core Concepts

## What is GuardDuty?

AWS GuardDuty is a continuous security monitoring service that analyzes and processes the following data sources:
- VPC Flow Logs
- DNS Logs
- AWS CloudTrail Event Logs
- Kubernetes Audit Logs
- S3 Data Events

## Key Components

### Detector
A single detector per region per account. It's the main resource that represents your GuardDuty service in a specific region.

### Filter
Used to categorize, suppress, or take action on specific findings. Filters can be used to:
- Archive low-severity findings
- Alert on high-severity findings
- Ignore known safe IP addresses

### IP Set
A list of trusted or malicious IP addresses that GuardDuty uses to evaluate findings.

### Threat Intel Set
A list of known malicious IP addresses, domains, or URLs that GuardDuty uses to detect threats.

### Findings
Generated by GuardDuty when it detects suspicious activity. Each finding includes:
- Severity level
- Resource affected
- Activity observed
- Time of detection

### Admin Account
A delegated administrator account for GuardDuty in an organization.

### Member Account
An AWS account that's part of an organization and monitored by the delegated administrator account.

## Service Architecture

1. **Data Collection**: GuardDuty collects and analyzes data from multiple AWS services
2. **Threat Detection**: Uses machine learning and threat intelligence to identify suspicious activity
3. **Finding Generation**: Creates findings for detected threats
4. **Alerting & Automation**: Sends alerts to Amazon EventBridge, Security Hub, or custom integrations

## Key Terms

- **Finding**: A specific detected threat or suspicious activity
- **Severity**: A score from 1.0 (low) to 10.0 (critical) indicating the severity of the finding
- **Detector ID**: Unique identifier for a GuardDuty instance in a region
- **Filter**: Rule that defines actions to take on matching findings
